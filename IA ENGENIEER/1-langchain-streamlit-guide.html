¿Qué es LangChain?

LangChain es un framework de Python que te permite construir aplicaciones con modelos de lenguaje (LLMs) de forma modular. En vez de hablar con la API de OpenAI o Anthropic directamente, LangChain te da piezas que encajan entre sí: prompts, cadenas, agentes, memoria y herramientas.

Analogía para entender

Imagina que un LLM es un chef muy talentoso. LangChain es la cocina completa: los utensilios, las recetas, el refrigerador (memoria), y los proveedores de ingredientes (APIs). Sin LangChain, tienes al chef solo con sus manos. Con LangChain, tiene todo para cocinar platos complejos.

Lo importante es que LangChain no es un modelo de IA — es el pegamento que conecta modelos con datos, herramientas y lógica de negocio.

🏗️

Arquitectura de Paquetes

LangChain se dividió en módulos separados para que solo instales lo que necesitas. Haz clic en cada capa para ver qué contiene:

📱 Tu Aplicación
Tu código, tu chatbot, tu agente
🔗 langchain
Chains, Agents, Memory — funcionalidad de alto nivel
langchain-core
Abstracciones base, LCEL
langchain-community
Integraciones terceros
Partner Packages
OpenAI, Anthropic, etc.

🔹 Tu aplicación es donde combinas todo. Puede ser un chatbot con Streamlit, un agente RAG, un pipeline de datos con IA, etc. Aquí importas lo que necesitas de las capas inferiores.

🔹 langchain tiene la lógica de alto nivel: ConversationChain, agentes que usan herramientas, memoria del chat, y callbacks. Es lo que "orquesta" todo.

🔹 langchain-core son los cimientos. Contiene las clases abstractas (BaseLanguageModel, Runnable), las plantillas de prompts, y el LCEL (LangChain Expression Language) para componer cadenas con |.

🔹 langchain-community conecta con herramientas de terceros: FAISS para vectores, loaders de PDF, conectores de bases de datos, etc. Son integraciones mantenidas por la comunidad.

🔹 Partner Packages son paquetes oficiales optimizados: langchain-openai, langchain-anthropic, langchain-ollama. Tienen mejor soporte y se actualizan independientemente.

¿Por qué se separó en módulos?

Antes todo venía en un solo paquete gigante. Si solo usabas OpenAI, igual instalabas dependencias de 50 proveedores más. Ahora es como un menú a la carta: solo pides lo que necesitas y pagas (en dependencias) solo por eso.

🪶

Instalación ligera

Solo instalas lo que usas. ¿Solo OpenAI? → langchain-core + langchain-openai

🔄

Updates independientes

Cada partner package se actualiza sin esperar un release del paquete principal.

🧹

Menos conflictos

Las dependencias de una integración no rompen otra. Cada paquete vive aislado.

🛠️

Mejor mantenimiento

Cada integración puede evolucionar a su propio ritmo con su propio equipo.

Imports y Ejemplos Prácticos

Cada import te dice de dónde viene la pieza que estás usando. Esto es clave para saber qué instalar:

Python 
# 1. Abstracciones base → langchain-core
from langchain_core.prompts import ChatPromptTemplate

# 2. Funcionalidad de alto nivel → langchain
from langchain.chains import ConversationChain

# 3. Integración específica → partner package
from langchain_openai import ChatOpenAI

# 4. Herramienta de terceros → community
from langchain_community.vectorstores import FAISS
Import desde... Paquete ¿Qué obtienes? Instalar
langchain_core.* core Prompts, Runnables, LCEL, interfaces base pip install langchain-core
langchain.* langchain Chains, Agents, Memory pip install langchain
langchain_openai.* partner ChatOpenAI, embeddings de OpenAI pip install langchain-openai
langchain_community.* community FAISS, loaders, herramientas varias pip install langchain-community
⚡
Regla de oro Si el import dice langchain_openai (con guión bajo y nombre del proveedor), es un partner package. Si dice langchain_community, es una integración comunitaria. Si dice langchain_core, es una abstracción fundamental.

¿Qué es Streamlit?

Streamlit convierte tu script de Python en una aplicación web interactiva sin necesidad de saber HTML, CSS ni JavaScript. Es el puente más rápido entre tu lógica de LangChain y una interfaz visual para el usuario.

Analogía para entender

Si LangChain es el motor del auto (la lógica, el cerebro), Streamlit es la carrocería + tablero: lo que el usuario ve y toca. Tu script Python se convierte directamente en botones, gráficos y chats.

Los 3 conceptos clave

No hay archivos HTML separados. El orden en que escribes las funciones en tu .py es el orden en que aparecen en pantalla. Arriba en el código = arriba en la web.

Python
import streamlit as st

st.title("Mi Chatbot")        # Aparece primero
st.write("Bienvenido 👋")     # Aparece segundo
pregunta = st.chat_input("Escribe algo...") # Aparece al final

Cuando el usuario hace clic o escribe algo, el script completo se ejecuta de nuevo de arriba a abajo. Esto simplifica el desarrollo pero tiene una consecuencia importante: las variables normales se pierden.

Usuario escribe "Hola"
↓
Streamlit re-ejecuta app.py completo
↓
Se renderiza la página con los nuevos datos
↓
Variables locales se pierden 💨
Por eso necesitas st.session_state

st.session_state es un diccionario especial que persiste entre ejecuciones. Es donde guardas el historial del chat, configuraciones del usuario, y cualquier dato que no quieras perder.

Python
# Inicializar historial (solo la primera vez)
if "messages" not in st.session_state:
    st.session_state["messages"] = []

# Agregar mensaje (persiste entre reruns)
st.session_state["messages"].append({
    "role": "user",
    "content": "Hola!"
})

⚡ Optimización

Python
# Cachear datos (se recalculan solo si cambian)
@st.cache_data
def cargar_csv(path):
    return pd.read_csv(path)

# Cachear recursos pesados (se cargan UNA sola vez)
@st.cache_resource
def cargar_modelo():
    return ChatOpenAI(model="gpt-4")
💡
@cache_data vs @cache_resource Usa @cache_data para datos (DataFrames, CSVs, resultados de cálculos). Usa @cache_resource para objetos globales pesados (modelos de ML, conexiones a BD). La diferencia: cache_data serializa una copia, cache_resource reutiliza el mismo objeto.

Recomendaciones

LangChain

✅ SÍ hacer
  • Instalar solo los paquetes que necesitas
  • Usar partner packages sobre community cuando existan
  • Aprender LCEL (el pipe |) — es el futuro de LangChain
  • Fijar versiones en requirements.txt
  • Empezar con cadenas simples antes de crear agentes
❌ NO hacer
  • Instalar langchain[all] — trae dependencias innecesarias
  • Importar desde langchain.llms (deprecated) — usar partner packages
  • Mezclar versiones incompatibles de core y partners
  • Hardcodear API keys en el código
  • Crear agentes complejos sin entender primero las cadenas

Streamlit

✅ SÍ hacer
  • Usar st.session_state para el historial del chat
  • Guardar API keys en .streamlit/secrets.toml
  • Cachear modelos con @st.cache_resource
  • Usar st.sidebar para configuración
  • Agregar .streamlit/ al .gitignore
❌ NO hacer
  • Poner API keys directamente en el código
  • Olvidar que cada interacción re-ejecuta todo el script
  • Crear el modelo LLM dentro del flujo principal (se recrea cada rerun)
  • Ignorar el caché — tu app será lenta sin él
  • Subir secrets.toml al repositorio
🔐
Seguridad — nunca olvidar Nunca pongas API keys en el código. Usa st.secrets["OPENAI_API_KEY"] con un archivo .streamlit/secrets.toml local, o variables de entorno en producción. Y asegúrate de que .streamlit/ esté en tu .gitignore.
