El corazón de cualquier sistema RAG. Aprende qué son, cómo funcionan y cuándo usar cada uno con ejemplos prácticos.

¿Qué es un Retriever?

Es la interfaz que recibe tu pregunta en lenguaje natural y devuelve los documentos más relevantes. No almacena documentos — solo los recupera.


Entrada: string de consulta
Salida: List[Document]

¿Por qué no usar la BD vectorial directamente?

La BD vectorial solo busca vectores cercanos. El retriever añade inteligencia: cuántos documentos traer, cómo reformular la pregunta, combinar fuentes, filtrar por relevancia...


Son compatibles con LCEL y se encadenan con cualquier chain.

Los 6 Retrievers más importantes
🔀
01 · con IA
MultiQuery
Retriever
Genera múltiples versiones de tu pregunta para recuperar más
+

Usa un LLM para crear 3–4 variaciones de tu consulta original y las lanza todas. Combina los resultados únicos. Soluciona el problema de que una sola búsqueda puede perderse información relevante con diferente terminología.

Úsalo cuando…
Tu consulta puede interpretarse de múltiples formas
La terminología del usuario difiere a la de los docs
Quieres mejorar el recall (no perderte nada)
Python
from langchain.retrievers.multi_query import MultiQueryRetriever
from langchain_openai import ChatOpenAI

llm = ChatOpenAI(temperature=0)
retriever = MultiQueryRetriever.from_llm(
    retriever=vectorstore.as_retriever(),
    llm=llm
)

# Una pregunta → 4 versiones → resultados combinados
docs = retriever.invoke("¿cuándo termina el contrato?")
🗜️
02 · con IA
Contextual
Compression
Recupera docs y los recorta a solo la parte relevante
+

Problema: los chunks recuperados suelen tener mucho texto irrelevante alrededor de la información útil. Este retriever primero recupera, luego usa un LLM para extraer solo el fragmento exactamente pertinente a tu pregunta.

Úsalo cuando…
Tus documentos son largos y con mucho ruido
Quieres reducir tokens enviados al LLM (ahorra dinero)
Necesitas respuestas muy precisas y citables
Python
from langchain.retrievers import ContextualCompressionRetriever
from langchain.retrievers.document_compressors import LLMChainExtractor

compressor = LLMChainExtractor.from_llm(llm)

retriever = ContextualCompressionRetriever(
    base_compressor=compressor,
    base_retriever=vectorstore.as_retriever()
)

# Devuelve solo el fragmento exactamente relevante
docs = retriever.invoke("¿cuál es la multa por impago?")
⚡
03 · híbrido
Ensemble
Retriever
Combina búsqueda semántica + búsqueda por palabras clave
+

Combina dos mundos: BM25 (busca palabras exactas, ideal para nombres propios o artículos legales específicos) + vectorial (busca por significado). Usa Fusión de Ranking Recíproco para combinar los resultados.

Úsalo cuando…
Tienes queries con términos técnicos exactos Y conceptuales
Documentos legales, médicos o técnicos (nomenclatura precisa)
Quieres el mejor resultado posible en producción
Python
from langchain.retrievers import EnsembleRetriever
from langchain_community.retrievers import BM25Retriever

bm25 = BM25Retriever.from_documents(docs)
bm25.k = 5
vector = vectorstore.as_retriever(search_kwargs={"k": 5})

# 30% keyword + 70% semántico
retriever = EnsembleRetriever(
    retrievers=[bm25, vector],
    weights=[0.3, 0.7]
)
🌲
04 · avanzado
ParentDocument
Retriever
Indexa pequeño, recupera grande
+

Divide los documentos en chunks pequeños (para embeddings precisos) pero cuando encuentra un match, devuelve el documento padre completo (para contexto suficiente). Lo mejor de chunks pequeños y grandes al mismo tiempo.

Úsalo cuando…
Chunks pequeños pierden contexto pero chunks grandes dan embeddings imprecisos
El LLM necesita párrafos completos para responder bien
Documentos con estructura clara (secciones, artículos)
Python
from langchain.retrievers import ParentDocumentRetriever
from langchain.storage import InMemoryStore

# Chunks pequeños → embedding más preciso
child_splitter = RecursiveCharacterTextSplitter(chunk_size=400)
# Chunks grandes → más contexto al recuperar
parent_splitter = RecursiveCharacterTextSplitter(chunk_size=2000)

retriever = ParentDocumentRetriever(
    vectorstore=vectorstore,
    docstore=InMemoryStore(),
    child_splitter=child_splitter,
    parent_splitter=parent_splitter,
)
🧠
05 · con IA
SelfQuery
Retriever
Convierte lenguaje natural en filtros estructurados automáticamente
+

Usa un LLM para interpretar tu consulta y generar filtros de metadata automáticamente. Si preguntas "películas de ciencia ficción después del 2010", crea el filtro genre=sci-fi AND year>2010 y luego busca semánticamente dentro de esos resultados.

Úsalo cuando…
Tus documentos tienen metadatos ricos (fecha, autor, categoría)
Usuarios hacen consultas mixtas (contenido + filtros)
Necesitas búsqueda estructurada sin que el usuario escriba SQL
Python
from langchain.retrievers.self_query.base import SelfQueryRetriever
from langchain.chains.query_constructor.base import AttributeInfo

metadata_fields = [
    AttributeInfo(name="year", description="Año", type="integer"),
    AttributeInfo(name="genre", description="Género", type="string"),
]

retriever = SelfQueryRetriever.from_llm(
    llm=llm,
    vectorstore=vectorstore,
    document_content_description="Películas",
    metadata_field_info=metadata_fields
)
# Crea filtros automáticamente desde lenguaje natural
docs = retriever.invoke("sci-fi después del 2010")
⏳
06 · básico
TimeWeighted
Retriever
Prioriza documentos más recientes, olvida los antiguos
+

Combina similitud semántica con recencia. Cada documento tiene un peso que decae con el tiempo según un decay_rate. Un documento muy relevante pero de hace meses puede quedar por debajo de uno moderadamente relevante de hoy.

Úsalo cuando…
Tienes datos que cambian con el tiempo (noticias, precios)
La información más reciente siempre es más válida
Construyes sistemas de memoria para agentes IA
Python
from langchain.retrievers import TimeWeightedVectorStoreRetriever

retriever = TimeWeightedVectorStoreRetriever(
    vectorstore=vectorstore,
    decay_rate=0.999,  # Qué tan rápido "olvida"
    k=5
)

# Documentos recientes tienen ventaja automática
docs = retriever.invoke("última actualización de precios")

Claves para elegir bien
01
Entiende tu caso de uso
Cada retriever tiene una fortaleza específica. No hay uno universalmente mejor.
02
Combina técnicas
Los mejores sistemas combinan varios retrievers. MultiQuery + Ensemble es una combinación ganadora.
03
Considera el costo
Los que usan LLM internamente (MultiQuery, Compression) hacen llamadas extra a la API. Evalúa el trade-off.
04
Mide la calidad
La calidad del retriever impacta directamente en la respuesta final. Un buen retrieval = buena respuesta.
