Conceptos fundamentales

Haz clic en cada concepto para expandir la explicación.

¿Qué es LangChain?›
LangChain es un framework en Python que te ayuda a construir aplicaciones con LLMs (Large Language Models). En vez de llamar APIs crudas, te da bloques reutilizables llamados componentes que se conectan entre sí.

Analogía: Si el LLM es el motor, LangChain es el chasis del auto: estructura, dirección y conexiones incluidas.
¿Qué es un ChatModel?›
Un ChatModel es el componente que representa al LLM. LangChain tiene adaptadores para Anthropic, OpenAI, Google, etc. Todos comparten la misma interfaz, así puedes cambiar de modelo sin reescribir tu código.

model = ChatAnthropic(
    model="claude-3-5-haiku-20241022",
    temperature=0.7
)
¿Qué es un PromptTemplate?›
Una plantilla de texto con variables dinámicas. En vez de hardcodear el prompt, defines el esqueleto y pasas los datos después.

prompt = ChatPromptTemplate.from_messages([
    ("system", "Eres un asistente útil."),
    ("human", "{pregunta}"),  # ← variable dinámica
])
Cuando invocas la chain, {pregunta} se reemplaza con el texto del usuario.
¿Qué es una Chain?›
Una secuencia de pasos conectados. Cada componente transforma la entrada y pasa el resultado al siguiente.

En LangChain moderno (LCEL), se construye con el operador | (pipe): 
chain = prompt | model | parser
El flujo es: PromptTemplate → ChatModel → OutputParser.
¿Qué es un OutputParser?›
El LLM devuelve un objeto AIMessage con metadata. El StrOutputParser extrae solo el texto limpio:

from langchain_core.output_parsers import StrOutputParser
parser = StrOutputParser()
# Convierte AIMessage → str

Cómo funciona la chain

Haz clic en cada bloque para ver qué hace en ese paso.

Input del usuario dict con variables
→
PromptTemplate formatea el mensaje
→
ChatAnthropic llama a la API
→
StrOutputParser extrae el texto
→
Respuesta final str limpio
Input: Un diccionario Python con las variables del prompt.
chain.invoke({"pregunta": "¿Qué es Python?"})
PromptTemplate recibe el dict, rellena {pregunta} y arma los mensajes:
[
  SystemMessage("Eres un asistente útil."),
  HumanMessage("¿Qué es Python?")
]
ChatAnthropic envía los mensajes a la API de Claude y recibe:
AIMessage(
  content="Python es un lenguaje...",
  response_metadata={...}
)
StrOutputParser extrae solo el texto del objeto AIMessage:
"Python es un lenguaje de programación..."
Resultado final: un str limpio listo para mostrar al usuario.
print(f"Asistente: {respuesta}")

El operador pipe |

LCEL (LangChain Expression Language) usa el mismo operador que la terminal Linux para componer componentes:

chain = prompt | model | parser
# Equivale a:
# parser(model(prompt(input)))

Cada componente implementa el protocolo Runnable, que garantiza métodos como .invoke(), .stream() y .batch().

Código y explicación

Lee el código completo y luego usa la sección interactiva para entender cada bloque.

# main.py

from dotenv import load_dotenv
from langchain_anthropic import ChatAnthropic
from langchain_core.prompts import ChatPromptTemplate
from langchain_core.output_parsers import StrOutputParser

# 1. Cargar variables de entorno
load_dotenv()

# 2. Definir el modelo
model = ChatAnthropic(
    model="claude-3-5-haiku-20241022",
    temperature=0.7,
)

# 3. Definir el prompt
prompt = ChatPromptTemplate.from_messages([
    ("system", "Eres un asistente útil y conciso."),
    ("human", "{pregunta}"),
])

# 4. Parser de salida
parser = StrOutputParser()

# 5. Construir la chain
chain = prompt | model | parser

# 6. Loop del chat
def main():
    print("🦜 Chat Simple con LangChain")
    while True:
        pregunta = input("Tú: ").strip()
        if not pregunta: continue
        if pregunta.lower() == "salir": break
        respuesta = chain.invoke({"pregunta": pregunta})
        print(f"Asistente: {respuesta}\n")

if __name__ == "__main__":
    main()

Explicación por bloques

load_dotenv()

Lee el archivo .env y carga ANTHROPIC_API_KEY como variable de entorno del sistema. LangChain la detecta automáticamente sin que debas pasarla explícitamente.

model = ChatAnthropic(model="claude-3-5-haiku-20241022", temperature=0.7)

Crea el objeto del ChatModel. Haiku es rápido y económico, ideal para aprender. temperature=0.7 da un balance entre precisión y naturalidad.

prompt = ChatPromptTemplate.from_messages([...])

Define los mensajes como lista de tuplas (rol, contenido). El rol system le da personalidad al modelo. {pregunta} es la variable dinámica.

chain = prompt | model | parser

La línea más importante. El operador | (pipe) conecta los tres componentes. El output de cada uno es el input del siguiente.

chain.invoke({"pregunta": pregunta})

Ejecuta toda la chain. El diccionario provee los valores para las variables del PromptTemplate. Retorna el string con la respuesta.

Errores comunes y soluciones

AuthenticationError: invalid x-api-key
Causa: La API key no está cargada correctamente.
Solución: Verifica que .env existe y tiene ANTHROPIC_API_KEY=sk-ant-... sin comillas ni espacios extra.
ModuleNotFoundError: No module named 'langchain_anthropic'
Causa: Dependencia no instalada en el entorno activo.
Solución: Activa el venv y ejecuta pip install langchain-anthropic.
KeyError: 'pregunta'
Causa: El nombre de la variable en invoke() no coincide con el del prompt.
Solución: El key del dict en chain.invoke({"pregunta": ...}) debe ser idéntico al de {pregunta} en el PromptTemplate.
La respuesta es un objeto, no texto
Causa: Falta StrOutputParser() al final de la chain.
Solución: chain = prompt | model | parser — no olvides el parser.
RateLimitError
Causa: Excediste el límite de requests de la API.
Solución: Espera unos segundos. En producción, agrega time.sleep(1) o retry logic.

Autoevaluación — responde y verifica

¿Qué hace el operador | en chain = prompt | model | parser?›
Conecta componentes en secuencia (LCEL). El output de cada componente se convierte en el input del siguiente, formando un pipeline de izquierda a derecha.
¿Por qué usamos StrOutputParser()?›
Porque el ChatModel devuelve un objeto AIMessage con metadata, no un string puro. El parser extrae solo el texto de la respuesta.
¿Por qué la API key va en .env y no en el código?›
Por seguridad. Si hardcodeas la key y subes el proyecto a GitHub, quedaría expuesta públicamente. El .env se excluye con .gitignore.
¿Qué significa que LangChain sea "agnóstico al modelo"?›
Que puedes cambiar el LLM (de Claude a GPT-4, a Gemini) con mínimos cambios, porque todos los ChatModels comparten la misma interfaz. Tu chain, prompt y parser siguen funcionando.
¿Qué recibe chain.invoke() y qué retorna?›
Recibe un diccionario con las variables del PromptTemplate (ej: {"pregunta": "..."}) y retorna el resultado final después de pasar por toda la chain — en este caso, un str con la respuesta.
