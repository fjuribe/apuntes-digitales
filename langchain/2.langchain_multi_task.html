LangChain Multi-Task con PromptTemplates

Un menú interactivo en consola donde el usuario elige una tarea y LangChain construye el prompt dinámicamente usando variables de entrada.

Teoría — PromptTemplate y variables

¿Qué es un PromptTemplate?

Un PromptTemplate es una plantilla de texto con marcadores de posición (variables) que se rellenan dinámicamente al momento de ejecutar la chain. Separa el qué decirle al LLM de los datos concretos del usuario.

Sin PromptTemplate
prompt = "Traduce '" + texto + "' al " + idioma

# Inflexible, propenso a errores,
# difícil de mantener
Con PromptTemplate
template = "Traduce '{texto}' al {idioma}"
prompt = PromptTemplate(
input_variables=["texto","idioma"],
template=template
)

Variables de entrada

Son los marcadores {variable} dentro del template. Al invocar la chain, se pasa un diccionario con sus valores:

chain.invoke({
    "texto": "Hello world",
    "idioma": "español"
})
LangChain valida automáticamente que todas las variables declaradas en input_variables estén presentes al momento de invocar. Si falta alguna, lanza un error claro.

Cómo LangChain separa prompt y lógica

Este es uno de los principios de diseño más importantes de LangChain. El prompt vive en su propio archivo o módulo, y la lógica del programa no lo contamina:

prompts/tareas.py

Solo define los templates y sus variables. Cero lógica de negocio.

main.py

Importa los prompts, construye las chains y maneja el menú. No sabe nada sobre cómo está redactado el prompt.

Esto te permite cambiar completamente un prompt sin tocar la lógica, y viceversa.

ChatPromptTemplate vs PromptTemplate

PromptTemplate ChatPromptTemplate
Un solo bloque de texto plano Lista de mensajes (system + human + assistant)
Para modelos completion clásicos Para modelos de chat (Claude, GPT-4...)

En este proyecto usamos ChatPromptTemplate porque Claude es un chat model.

# prompts/tareas.py
# Este archivo es SOLO prompts. Cero lógica de negocio.

from langchain_core.prompts import ChatPromptTemplate

# ── Tarea 1: Resumir ─────────────────────────────────────
prompt_resumir = ChatPromptTemplate.from_messages([
    ("system", "Eres un asistente experto en síntesis de textos."),
    ("human", "Resume el siguiente texto en {largo} oraciones:\n\n{texto}"),
])

# ── Tarea 2: Traducir ────────────────────────────────────
prompt_traducir = ChatPromptTemplate.from_messages([
    ("system", "Eres un traductor profesional y preciso."),
    ("human", "Traduce el siguiente texto al {idioma}:\n\n{texto}"),
])

# ── Tarea 3: Explicar concepto ───────────────────────────
prompt_explicar = ChatPromptTemplate.from_messages([
    ("system", "Eres un profesor experto en explicar conceptos complejos de forma simple."),
    ("human", "Explícame qué es '{concepto}' de forma clara para alguien con nivel {nivel}. Usa ejemplos concretos."),
])

# ── Tarea 4: Redactar correo ─────────────────────────────
prompt_correo = ChatPromptTemplate.from_messages([
    ("system", "Eres un asistente experto en comunicación profesional y redacción de correos."),
    ("human", "Redacta un correo con tono {tono} sobre el siguiente asunto: {asunto}\n\nDetalles adicionales: {detalles}"),
])


  
from langchain_core.prompts import ChatPromptTemplate
Importamos la clase para crear templates de chat (con roles system/human).

  
ChatPromptTemplate.from_messages([...])
Recibe una lista de tuplas (rol, contenido). El contenido puede tener variables {variable}.

  
"{largo} oraciones ... {texto}"
Dos variables en el mismo template. Al invocar, debes proveer ambas en el diccionario.


  
Archivo main.py

  
# main.py

from dotenv import load_dotenv
from langchain_anthropic import ChatAnthropic
from langchain_core.output_parsers import StrOutputParser
from prompts.tareas import (
    prompt_resumir, prompt_traducir,
    prompt_explicar, prompt_correo
)

# ── Setup ────────────────────────────────────────────────
load_dotenv()
model  = ChatAnthropic(model="claude-3-5-haiku-20241022", temperature=0.5)
parser = StrOutputParser()

# ── Función genérica de ejecución ───────────────────────
def ejecutar(prompt_template, variables: dict) -> str:
    chain = prompt_template | model | parser
    return chain.invoke(variables)

# ── Tarea 1 ──────────────────────────────────────────────
def tarea_resumir():
    texto = input("Ingresa el texto a resumir:\n> ")
    largo = input("¿En cuántas oraciones? [default: 3]: ") or "3"
    return ejecutar(prompt_resumir, {"texto": texto, "largo": largo})

# ── Tarea 2 ──────────────────────────────────────────────
def tarea_traducir():
    texto  = input("Ingresa el texto a traducir:\n> ")
    idioma = input("¿A qué idioma? [default: inglés]: ") or "inglés"
    return ejecutar(prompt_traducir, {"texto": texto, "idioma": idioma})

# ── Tarea 3 ──────────────────────────────────────────────
def tarea_explicar():
    concepto = input("¿Qué concepto quieres entender?\n> ")
    nivel    = input("Nivel [básico/intermedio/experto, default: básico]: ") or "básico"
    return ejecutar(prompt_explicar, {"concepto": concepto, "nivel": nivel})

# ── Tarea 4 ──────────────────────────────────────────────
def tarea_correo():
    asunto   = input("Asunto del correo:\n> ")
    tono     = input("Tono [formal/informal/amigable, default: formal]: ") or "formal"
    detalles = input("Detalles adicionales:\n> ")
    return ejecutar(prompt_correo, {"asunto": asunto, "tono": tono, "detalles": detalles})

# ── Menú principal ───────────────────────────────────────
TAREAS = {
    "1": ("Resumir texto",     tarea_resumir),
    "2": ("Traducir texto",    tarea_traducir),
    "3": ("Explicar concepto", tarea_explicar),
    "4": ("Redactar correo",   tarea_correo),
}

def main():
    print("🛠️  LangChain Multi-Task\n")
    while True:
        print("\nElige una tarea:")
        for k, (nombre, _) in TAREAS.items():
            print(f"  {k}. {nombre}")
        print("  0. Salir")

        opcion = input("\nOpción: ").strip()
        if opcion == "0": break
        if opcion not in TAREAS:
            print("Opción inválida.")
            continue

        _, fn = TAREAS[opcion]
        print("\n⏳ Procesando...\n")
        resultado = fn()
        print(f"\n{'─'*50}\n{resultado}\n{'─'*50}")

if __name__ == "__main__":
    main()


  
def ejecutar(prompt_template, variables)
Función genérica que construye la chain y la invoca. Evita repetir prompt | model | parser en cada tarea.

  
TAREAS = {"1": ("nombre", funcion), ...}
Diccionario que mapea opciones del menú a funciones. Permite agregar tareas fácilmente sin modificar el loop principal.

  
_, fn = TAREAS[opcion]
Desestructura la tupla. El _ descarta el nombre (ya fue impreso), fn es la función que se ejecuta.

  
or "3" / or "formal"
Valores por defecto usando el operador or: si el usuario no escribe nada, se usa el valor predeterminado.

Buenas prácticas al diseñar prompts


  
1. Sé específico en el rol del sistema

  

    
Vago
"Eres un asistente útil."

    
Específico
"Eres un traductor profesional especializado en textos técnicos y legales."

  


  
2. Usa variables para todo lo que cambia

  

    
Hardcodeado
"Resume en 3 oraciones el texto."

    
Variable
"Resume en {largo} oraciones el texto."

  


  
3. Da formato a la salida esperada

  

    
Especifica el formato de respuesta cuando importa

    
Si quieres bullet points, una lista numerada o JSON, dilo explícitamente en el prompt. No asumas que el LLM adivinará tu preferencia.

  

  
"Explica {concepto}. Usa este formato:
1. Definición simple (1 oración)
2. Analogía del mundo real
3. Ejemplo práctico en código"


  
4. Separa instrucciones de datos

  

    
Instrucciones arriba, datos del usuario abajo

    
Poner los datos del usuario al final reduce la posibilidad de que el modelo confunda instrucciones con contenido a procesar.

  

  
"Traduce el siguiente texto al {idioma}.
Mantén el tono original.
No añadas explicaciones.

Texto a traducir:
{texto}"


  
5. Errores comunes en diseño de prompts


  

    
Prompt ambiguo

    
Causa: instrucciones vagas que el LLM interpreta de formas distintas.

    
Solución: "Responde solo con el texto traducido, sin explicaciones adicionales."

  

  

    
Demasiadas instrucciones mezcladas

    
Causa: prompt con 10 reglas distintas en un párrafo.

    
Solución: usa listas numeradas o viñetas dentro del template para separar instrucciones.

  

  

    
Variables sin contexto suficiente

    
Causa: {texto} sin ninguna instrucción sobre qué hacer con él.

    
Solución: siempre acompaña las variables con la instrucción explícita de qué hacer.

  


  
Un buen prompt no es el más largo — es el más claro. La claridad supera la verbosidad.


  






  
Cómo escalar este proyecto

  
Este proyecto básico tiene una arquitectura que escala naturalmente. Aquí el camino de crecimiento:


  

    Nivel 2
    

      
Agregar más tareas sin tocar el menú

    Nivel 4
    

      
Interfaz web con Streamlit

      
Reemplazar el menú de consola por una UI con selectbox y text_area de Streamlit. La lógica de prompts no cambia en absoluto.

API REST con FastAPI

      
Exponer cada tarea como un endpoint POST /resumir, POST /traducir, etc. El cuerpo del request provee las variables del prompt.

Agregar una tarea "Consultar documento" que carga un PDF, lo indexa con FAISS y responde preguntas sobre su contenido usando RetrievalQA.

Autoevaluación

  

    
¿Qué ventaja tiene separar los prompts en su propio archivo?›

    
Permite cambiar el wording de un prompt sin tocar la lógica del programa, y viceversa. Mejora la mantenibilidad, la legibilidad y facilita probar distintas versiones de un prompt.

  

  

    
¿Por qué usamos una función ejecutar() genérica en lugar de repetir la chain en cada tarea?›

    
Para aplicar el principio DRY (Don't Repeat Yourself). La lógica de construir la chain es siempre prompt | model | parser — centralizarla evita bugs, facilita cambios y hace el código más limpio.

  

  

    
¿Qué pasaría si invocas la chain sin una de las variables declaradas en el PromptTemplate?›

    
LangChain lanzaría un KeyError o un error de validación indicando qué variable falta. Esto es un comportamiento intencional — te fuerza a proveer todos los datos que el prompt necesita.

  

  

    
¿Por qué el patrón de diccionario TAREAS es mejor que un if/elif largo?›

    
Con el diccionario, agregar una tarea nueva es una sola línea. Con if/elif, habría que editar el bloque de control cada vez. El diccionario también permite iterar las opciones para imprimir el menú automáticamente.

  

  

    
¿Cómo cambiarías el modelo de Claude a GPT-4 sin modificar los prompts?›

    
Solo cambia la línea model = ChatAnthropic(...) por model = ChatOpenAI(model="gpt-4o", ...) en main.py. Los prompts, las chains y las tareas quedan exactamente igual, porque todos los ChatModels comparten la misma interfaz en LangChain.
